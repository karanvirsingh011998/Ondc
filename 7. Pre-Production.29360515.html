7. Pre-Production

Pre-Production is the  environment where Network Participants test their development to receive certification to move to ONDC Production Environment.

The process for entry to pre-production registry can be found in the following document:

https://github.com/ONDC-Official/developer-docs/blob/main/registry/Onboarding%20of%20Participants.md

API Integration

For Pre-Production, the network participants need to complete the end-to-end testing with ONDC reference applications and two different Network participants who are present in pre-production registry.

Being a Buyer Network participant, kindly test the end-to-end transaction flow with https://ref-seller-app-preprod.ondc.org/.

Being a seller Network participant, kindly test the end-to-end transaction flow with https://buyer-app-preprod.ondc.org/login.

  • Operational Compliance:

Common for Buyer Node, Seller Node and Technology Service Providers

1

Compliance (Process | Description)

Status

2

Customer Support Implementation (GRM)
Network Participants must set up a customer support team to deal with any returns, refunds, customer grievances etc. Refer to the document provided for details.

3

Reconciliation and Payment Settlement
Network Participants must set up a Reconciliation and Payment Settlement Mechanism.
In the initial phase, only buyer applications will be collecting payments and COD orders will not be entertained. As such, for Beta Launch, all Buyer applications must have a Reconciliation and Payment Settlement mechanism in place.
Refer to the document provided for details.

4

Support to Delivery Agents
Network Participants (Seller Nodes and Logistics Nodes) must set up a process for supporting delivery agents to deliver orders in a prompt and safe manner.
Refer to the document provided for details.

5

Process for Monitoring of KPIs & SLAs
Certain service level commitments are expected from Network Participants. These are:
Transaction drop rate (<1%)

Seller Cancellations (<5%)

Transaction drop rate (<5%)

Delivery within committed duration (>95%)

Participants are expected to track the aforementioned KPIs through an Order Status Dashboard and should ensure that the commitments are met. Daily monitoring of the dashboard will be conducted by ONDC to identify breach of SLAs for order execution/ customer support. Refer to the document provided for more details.
